# Decision Records

This directory contains short architecture decision records.

Create a decision record when a choice affects future implementation, security posture, dependencies, testing strategy, or workflow.

Do not create decision records for obvious local implementation details.

## Format

Each record should include:

- Context.
- Decision.
- Consequences.
- Rejected alternatives.

Decision records are append-only by default. If a decision changes, add a new record that supersedes the previous one.
