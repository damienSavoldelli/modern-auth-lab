# Security Concepts

This directory contains reusable technical explanations for authentication concepts used by Modern Auth Lab.

These documents are written for two purposes:

- guide implementation decisions inside the project;
- provide structured material for future technical articles.

## Current Concept Notes

- [TOTP](totp.md)
- [Passkeys and WebAuthn](passkeys.md)
- [MFA fallback](mfa-fallback.md)
- [Trusted devices](trusted-devices.md)
- [Sources](sources.md)

## Writing Rules

Concept notes should explain:

- the user-facing goal;
- the security objective;
- the assets being protected;
- the attacker capabilities considered;
- the trust boundaries;
- the implementation implications;
- the UX/security tradeoffs;
- what is intentionally deferred.
